### 编译的过程及产物

执行命令之后发生了什么。

#### 命令 1

```bash
bash build.sh --opkernel --ops=index_put_with_sort_v2 --soc=ascend950
```

#### 1. 调用 cmake configure

对算子要编译的目录添加到 target 中，匹配规则就是：

- `tiling -> arch35`
- `infershape -> op_host`
- `def -> op_api`

只会在 `build` 下产生 `Makefile`。

#### 2. `cmake --build`

**Step 1：编译 `tiling` / `infershape` / `opdef` 动态库**

```text
g++ -std=c++17 -fvisibility=hidden -fno-strict-aliasing ...
  op_host/arch35/index_put_with_sort_v2_tiling_arch35.cpp
  -> build/libophost_ops_nn.so  (含 tiling 函数)
```

**Step 2：`opbuild_gen_aclnnExc` target**

调用 `op_build` 工具（CANN 的 `opbuild` 二进制）处理 `_def.cpp`，生成 `.ini` 算子信息文件，供后续 OPC 使用：

```text
op_build gen_op_host_aclnnExc.so -> build/tbe/config/ascend950/index_put_with_sort_v2.ini
```

**Step 3：`ascendc_impl_gen` target**

读取 `.ini` + kernel 源码，生成 Python 描述文件（动态 Python 文件），描述 kernel 的编译参数：

```text
build/tbe/index_put_with_sort_v2_xxx.py   <- asc_opc 的输入之一
```

**Step 4：`gen_bin_scripts` target**

调用 `build_binary_opc_gen_task.sh`，读取 `op_host/config/ascend950/index_put_with_sort_v2_binary.json`，按 dtype 变体生成 `opc_cmd.sh`，每行一条 `asc_opc` 命令：

```bash
# build/binary/ascend950/bin/opc_cmd/opc_cmd.sh（片段）
asc_opc build/tbe/index_put_with_sort_v2.py \
  --main_func=index_put_with_sort_v2 \
  --input_param=.../index_put_with_sort_v2_binary_high_performance_0 \
  --soc_version=Ascend950A \
  --output=build/binary/ascend950/index_put_with_sort_v2/ \
  --impl_mode=high_performance \
  --simplified_key_mode=0 \
  --op_mode=dynamic
```

**Step 5：`binary` target（多线程并行）**

`build.sh` 统计 `opc_cmd.sh` 行数 -> 传给 CMake `-DOPC_NUM_ascend950=N`，并行调用所有 `asc_opc` 命令。

`asc_opc`（本质是调用 `bisheng/CCE` 编译器）：

```text
.py kernel描述 + .json dtype参数
    ↓
AscendC C++ 源码 (.cpp)
    ↓  bisheng/CCE 编译 (-O2 --target=ascend950)
.o 二进制文件
    ↓
build/binary/ascend950/index_put_with_sort_v2/
  ├── IndexPutWithSortV2_float16.o
  ├── IndexPutWithSortV2_float32.o
  ├── IndexPutWithSortV2_bool.o
  └── ... (每个 dtype 一个 .o)
```

### `opapi_nn` target 编译什么

`opapi_nn` 是 `aclnn` 接口层的动态库，纯 CPU C++ 代码，与 SoC 无关。编译内容：

```text
所有算子的 op_api/aclnn_xxx.cpp
     ↓ opbuild_gen_aclnnExc（从 _def.cpp 生成）
aclnn_index_put_with_sort_v2.cpp（自动生成）+ 其他所有算子
     ↓ g++ -std=c++17
libopapi_nn.so
```

```text
opc_cmd.sh  -> 并行 N 线程 asc_opc 编译
     ↓ 全部完成后
out_cmd.sh  -> 串行/并行 gen_output_json.py 汇总
     ↓
build/binary/config/ascend950/*.json  (运行时索引)
```

### 编译的 kernel 产物：`.json` 和 `.o` 是干什么的

`.json` 和 `.o` 配对关系：

`asc_opc` 对每个 `bin_filename` 同时输出两个文件：

```text
build/binary/ascend950/bin/ascend950/index_put_with_sort_v2/
├── IndexPutWithSortV2_bool.o       <- NPU 机器码
├── IndexPutWithSortV2_bool.json    <- 匹配条件元数据
├── IndexPutWithSortV2_float16.o
├── IndexPutWithSortV2_float16.json
├── IndexPutWithSortV2_float32.o
├── IndexPutWithSortV2_float32.json
└── ...
```

`.json` 描述“什么条件下能用这个 kernel”，`.o` 是实际要跑的代码，必须成对才能工作。
